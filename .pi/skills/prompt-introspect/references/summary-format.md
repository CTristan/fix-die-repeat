# Introspection Summary Format

`~/.config/fix-die-repeat/introspection-summary.md` is a Markdown file generated/regenerated by the `prompt-introspect` skill. It captures cumulative patterns, lessons learned, and template changes applied across all processed PR review introspection data.

The skill uses this file for trend context during template updates, replacing raw YAML for historical analysis.

## Structure

```markdown
# Introspection Summary

Last updated: <date>
Total PRs analyzed: <count>
Projects: <list>

## Pattern Frequency
<!-- Ranked by occurrence count -->
- **<category>** (<count> threads): <description of common patterns>
- ...

## Key Lessons Learned
<!-- Actionable insights distilled from all reviewed entries -->
- <lesson>
- ...

## Template Changes Applied
<!-- Log of what was changed and why, most recent first. PRESERVE THIS SECTION ACROSS RUNS -->
- <date>: <template file> — <what changed> (triggered by <category/pattern>)
- ...

## Coverage Gaps
<!-- Categories or patterns not yet addressed by templates -->
- <gap description>
```

## Regeneration Guidelines

1. **Cumulative**: Incorporate context from the existing `introspection-summary.md` and any new `pending` entries.
2. **Concise**: Target under 200 lines to remain LLM-friendly.
3. **Structured**: Use standard Markdown headers and list items.
4. **Append-Only Log**: The "Template Changes Applied" section must be preserved and appended to across regenerations.
5. **Pattern Extraction**: Distill common thread details into high-level patterns rather than listing individual threads.
